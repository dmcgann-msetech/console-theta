Executive Directive

Theta has crossed from module-extraction work into foundation-restructure work. The app already has split JavaScript areas, Supabase-backed workflows, Cloudflare deployment files, backup automation, docs, tools, and root-level emergency artifacts. The real problem is that the split pieces still behave like one browser-global program.

The restructure will preserve the current DOM, page IDs, business workflows, Supabase schema, and deployment availability while introducing a real app shell, router, state store, services layer, module boundaries, reusable UI components, tests, and a controlled dist deployment path.

Directive: Do not split more index markup as the next move. Do not rewrite. Do not add a frontend framework. Build the foundation, then migrate domains one at a time.

2. Operating Principles

Preserve Product Stability

Current UI and workflows stay operational. Architecture is replaced underneath, not by a broad UI rewrite.

Strangle Legacy Runtime

Introduce app shell, store, and services beside the old system. Remove old paths only after the new path is proven.

One Narrow Job

Every commit must have one purpose, a validation step, a rollback path, and a roadmap/log update.

Non-Negotiables

  • No rewrite.
  • No React, Vue, Angular, or Electron.
  • No new production feature work until governance and build foundation are complete.
  • No direct Supabase calls from migrated UI modules.
  • No root artifact deletion until archive and restore drill are complete.
  • No Cloudflare dist switch until Vite build, smoke tests, and rollback are proven.

6. Validation and Release Gates

Required Before Every Merge to Master

powershell -ExecutionPolicy Bypass -File .\tools\scan-mojibake.ps1
git diff --check
npm run lint
npm run test -- --run
npm run test:e2e -- --grep @smoke
npm run build

Manual Smoke Checklist

  • Login screen loads.
  • Google sign-in starts correctly.
  • Dashboard opens.
  • Sidebar navigation opens every page.
  • Tickets: search, filters, open panel, create/edit flow.
  • Dispatch: queue, assignments, field status.
  • Clients: list, panel, forms picker, document list.
  • Documents: folders, refresh, edit attachment metadata, view/print.
  • Accounting: Overview, GL, A/R, A/P, Banking, Expenses, Reconciliation, Reports, Tax, Audit.
  • Inventory: Assets and Parts load.
  • HR / My Time Off load for authorized users.
  • Settings / Quick Links / Sign-In Log load for authorized users.
  • No fatal console errors.
  • No mojibake in visible navigation, buttons, forms, or document templates.

Definition of Done for Each Phase

  • Roadmap progress log updated.
  • Architecture decision recorded if a decision changed.
  • Build/test/smoke commands run and recorded.
  • Rollback path identified.
  • Git status clean.
  • Remote branch pushed.
